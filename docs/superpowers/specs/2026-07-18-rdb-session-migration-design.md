# RDB 会话迁移与合并验收设计

## 目标

修复从 Preferences 会话存储升级到 RDB 后，运行中或暂停的专注会话无法恢复的问题；完成合并功能的设备验收与 Git 收尾。

## 范围

- 将旧 Preferences 中的 `active_focus_session` 一次性迁移到 RDB 的 `active_session`。
- 迁移后由 RDB 会话驱动专注页和通知；旧会话不存在时取消 ID 为 `1001` 的遗留通知。
- 验收清空脑袋的添加、编辑、排序、设为下一件事、重启恢复、今日回顾，以及专注通知的运行、暂停、结束和重启恢复。
- 提交现有 Canva UI 计划文档，但不创建 Canva 设计。
- 清理临时验证工作区，并推送 `main`。

## 迁移规则

1. 应用启动且 RDB 已初始化后，先读取 RDB 活跃会话。
2. RDB 已有合法会话时保持原样，不读取或覆盖旧 Preferences 会话。
3. RDB 没有活跃会话时，读取旧 Preferences 会话。旧会话合法则转换为 `ActiveSession`：任务名称、时长、剩余秒数、目标结束时间和运行状态保持不变；结果页使用 `pendingRecordId = 0`，以便现有结果页流程创建待处理记录。
4. 仅在 RDB 保存成功后清除旧 Preferences 会话。
5. 未找到可迁移会话或旧会话无效时，取消遗留通知；无效旧会话沿用旧存储的清理行为。
6. 迁移本身不得创建或取消有效会话的新通知；恢复后的 `FocusTimer` 负责按现有逻辑发布或取消通知。

## 结构

- 新建纯函数模块负责校验和转换旧 `FocusSession` 到 `ActiveSession`，使关键规则可由 Hypium 测试。
- `Index` 在加载 RDB 活跃会话前执行迁移协调逻辑，并只在没有可恢复会话时取消旧通知。
- `FocusSessionStore` 继续作为旧版本数据的读取和清理边界，不参与新会话保存。

## 验收

使用 API 24 模拟器先安装迁移前构建并创建专注会话，再覆盖安装当前构建：

- 运行中会话按目标结束时间恢复，通知显示运行状态。
- 暂停会话保留剩余时间，通知显示暂停状态。
- 无旧会话时，通知中心不保留旧通知。
- 清空脑袋支持添加、编辑、手动排序、设为下一件事；重启后数据仍在。
- 已结束专注在今日回顾中呈现正确状态和统计。
- 结束专注后通知消失；重启运行中会话后通知恢复。

## 非目标

- 不迁移旧版本的历史专注记录，因为旧版本没有持久化历史记录。
- 不实现 Canva 设计，只将现有计划纳入版本控制。
