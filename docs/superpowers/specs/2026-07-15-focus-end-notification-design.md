# 专注结束提醒设计

## 目标

在专注结束时发送系统通知，即使应用切后台或被关闭也能提醒用户；点击通知直接进入本次专注的结果页。

## 行为

- 第一次开始专注时请求通知权限。拒绝权限不阻塞专注。
- 设置页有“专注结束提醒”开关，默认开启并保存到本地。
- 开始和继续专注时，若开关开启且权限有效，则根据目标结束时间安排一次系统通知。
- 暂停、提前结束、结果确认时取消通知；继续时重新安排。
- 前台归零直接进入结果页；不再额外弹出重复通知。
- 通知显示“专注时间到了”及任务名称。点击通知读取已保存会话并打开结果页，不重复创建专注记录。

## 实现边界

使用 HarmonyOS 系统定时提醒和通知 API，不引入常驻后台服务。通知权限、开关和提醒 ID 通过现有 RDB `app_state` 持久化。无权限或系统安排失败时记录中文反馈但保持专注流程可用。

## 验收

模拟器中允许通知权限后，启动短计时并切后台，确认结束通知出现；点击通知进入结果页。暂停、提前结束、完成或放弃后不再保留提醒。重启应用后，未结束会话仍有对应提醒；关闭后已完成会话不再提醒。
